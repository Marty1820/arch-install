## Taken from https://gist.github.com/koiyakiya/1e41226b7432eab24db1a5f0f5f607ff

1. Run `bootctl status` and make sure 'Secure Boot:' says Setup Mode. Run the commands if it is.

```
$ systemctl enable NetworkManager.service
$ sbctl create-keys
$ sbctl enroll-keys -m
$ sbctl verify
/boot/EFI/Linux/...efi
...
$ sbctl sign -s [file given as result of sbctl verify command]
$ sbctl sign -s [second file given as result of sbctl verify command]
$ sbctl sign -s ...
$ ...
```

Do this until you have signed every files as listed in `sbclt verify`

2. Reboot and relogin to root. Check `bootctl status` to see if secure boot says enabled. If it is, move on to the next step.
- Might say enable but you will have to enable in the UEFI too.

3. Enroll TPM 2.0 | change `nvme0n1p3` to your actual encrypted partition
```
$ systemd-cryptenroll /dev/nvme0n1p3 --recovery-key >> /.cryptkey
$ systemd-cryptenroll /dev/nvme0n1p3 --wipe-slot=tpm2 --tpm2-device=auto --tpm2-pcrs=0+2+4+7:sha256
```
Make sure to save the recovery key somewhere safe. It is very important.

4. Reboot and you are done. You shouldn't be asked for the decryption password if everything worked.
